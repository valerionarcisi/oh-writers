# 11 — Budget & Cost Prediction

## Overview

The budget module translates breakdown elements into estimated production costs. It does not replace a professional budget software (like Movie Magic Budgeting), but gives directors and producers a realistic, AI-assisted cost estimate early in pre-production — before it is too late to make structural decisions.

The AI uses genre, format, shooting days, and breakdown elements to generate a starting point. The user refines it.

---

## Core Concepts

### Budget Structure

A budget is organized in top sheets — high-level categories — each containing budget lines.

```
Budget
├── Above the Line
│   ├── Story & Rights
│   ├── Producer
│   ├── Director
│   └── Cast (principal)
├── Below the Line — Production
│   ├── Extras & Background
│   ├── Set Design & Art Dept.
│   ├── Props
│   ├── Costumes & Wardrobe
│   ├── Locations & Permits
│   ├── Vehicles & Transportation
│   ├── Camera & Lighting
│   ├── Sound
│   └── VFX / Special Effects
├── Below the Line — Crew
│   ├── Director of Photography
│   ├── First AD
│   ├── Production Designer
│   └── Department heads...
└── Contingency (% of total)
```

### Cost Types

| Type         | Description                                          |
| ------------ | ---------------------------------------------------- |
| `daily`      | Cost per shooting day (crew rates, equipment rental) |
| `flat`       | One-time fixed cost (location fee, rights purchase)  |
| `weekly`     | Weekly rate (some crew contracts)                    |
| `unit`       | Cost per unit (costume pieces, props)                |
| `percentage` | Percentage of subtotal (contingency, insurance)      |

### AI Estimate vs Actual

Every budget line has two values:

- `aiEstimate` — generated by AI, read-only reference
- `actual` — the value the user sets, used for all calculations

If `actual` is null, the AI estimate is used in totals. This means you get a complete budget from day one, which the user progressively refines.

## Data Model

```typescript
export const BudgetLineSchema = z.object({
  id: z.string().uuid(),
  budgetId: z.string().uuid(),
  topSheetCategory: z.enum([
    "above_the_line",
    "production",
    "crew",
    "post_production",
    "contingency",
  ]),
  name: z.string().min(1).max(200),
  costType: z.enum(["daily", "flat", "weekly", "unit", "percentage"]),
  quantity: z.number().nullable(), // days, units, weeks
  rate: z.number().nullable(), // cost per unit
  aiEstimate: z.number().nullable(), // total estimate from AI
  actual: z.number().nullable(), // user-set value, overrides aiEstimate
  notes: z.string().nullable(),
  linkedElementCategory: z
    .enum([
      // which breakdown category feeds this line
      "cast",
      "extras",
      "props",
      "costumes",
      "locations",
      "vehicles",
      "vfx",
      "sfx",
      "sound",
    ])
    .nullable(),
});

export const BudgetSchema = z.object({
  id: z.string().uuid(),
  projectId: z.string().uuid(),
  currency: z.string().default("EUR"),
  contingencyPercent: z.number().min(0).max(50).default(10),
  status: z.enum(["draft", "ai_estimated", "in_review", "locked"]),
  lines: z.array(BudgetLineSchema),
  generatedAt: z.date().nullable(),
});

export type Budget = z.infer<typeof BudgetSchema>;
export type BudgetLine = z.infer<typeof BudgetLineSchema>;
```

## Effective Value Rule

The effective value of a budget line is always:

```typescript
const effectiveValue = (line: BudgetLine): number => {
  if (line.actual !== null) return line.actual;
  if (line.aiEstimate !== null) return line.aiEstimate;
  return 0;
};

const lineTotal = (line: BudgetLine): number => {
  const value = effectiveValue(line);
  if (line.costType === "percentage") return 0; // handled separately
  if (line.quantity !== null && line.rate !== null)
    return line.quantity * line.rate;
  return value;
};
```

Contingency is always calculated as a percentage of the subtotal of all non-contingency lines.

## AI Cost Prediction

### Inputs

The AI receives:

- Project format (`feature`, `short`, `series_episode`, `pilot`)
- Project genre
- Estimated shooting days (from schedule or breakdown)
- Breakdown summary (element counts by category)
- Country/region (for crew rates baseline)

### What AI estimates

- Daily crew rates per department (based on Italian/EU industry averages)
- Equipment rental costs per shooting day
- Location fees based on location type (urban, rural, studio, exterior)
- Extras cost based on quantity × shooting days
- VFX line based on VFX scene count and complexity
- Cast line based on number of principal roles × estimated shooting days

### What AI does NOT estimate

- Story rights — too variable
- Producer and director fees — personal negotiation
- Insurance — requires actual quotes
- Post-production — deferred to a later feature

These lines are created as empty placeholders for the user to fill in.

### Prompt constraints

The AI must:

- Return itemized JSON matching `BudgetLine[]`, never a single total
- Include source reasoning for each estimate in a `reasoning` field (not stored, shown in UI during review)
- Flag lines with high uncertainty (e.g. VFX, special locations) with `confidence < 0.6`
- Use conservative estimates — better to be over than under

## User Flows

### Generate budget estimate

1. User opens Budget tab — requires breakdown to be at least partially confirmed
2. Prompt: "Generate AI cost estimate" — shows what inputs will be used
3. Loading state → budget lines appear grouped by top sheet
4. Each line shows: name, AI estimate, confidence indicator
5. User reviews line by line, setting `actual` values where they have real numbers

### Edit a budget line

- Click `actual` field → inline number input
- Tab to move to next line
- Lines with `actual` set show a "user" indicator, lines using AI estimate show "AI" indicator
- Notes field available for every line

### Budget summary

Always visible at the top:

- Total above the line
- Total below the line
- Subtotal
- Contingency (calculated)
- **Grand total**

Color coding:

- Green: AI estimate, not yet reviewed
- White: User-confirmed value
- Red: Line flagged as uncertain by AI
